一、需求概述

1、需求背景

   当前商户入网开通流程中，销售人员需要先收集商户的证照等资料和产品开通需求，然后提OA入网流程（填写商户资料和产品开通费率等信息），运营人员根据OA申请进行资质审核并手动填写纸质版协议后，再为商户开通产品和激活账户，期间纸质协议会在运营人员、销售人员和商户之间往来邮寄多次，商户的产品开通顺序也受到协议签署结果的影响，一般需等纸质协议归档后才能使用结算功能。
   针对以上问题，本次商户入网流程的优化主要集中在资料收集、产品开通、协议签署以及OA审核等环节，达到简化运营流程、平衡和减轻运营和销售人员工作量、实现商户入网签约流程标准化和线上化、提高入网签约效率的目的。

2、需求定义

2.1 关于产品开通

  • 商户入网签约到开通产品的整个体验过程可以分为

    • 账户注册
    • 实名认证
    • 产品开通

  • 产品开通也是一个独立的流程，产品的开通方式可以划分为以下几种

    • 协助开通：销售发起申请，提OA附带产品申请信息，运营审核开通，签署协议

    • 自助开通：商户发起申请，提交产品开通申请，运营审核开通，签署协议

      • 产品码开通：指定的产品和费率，可同时开通多个产品，通过代理商/销售获取产品开通码
      • 默认开通：默认的产品和费率
      • 普通开通：通用产品和费率，直接在线选择并提交开通申请
2.2 关于协议分类

  • 协助开通（目前公司的产品开通方式，使用的协议如下）

    • 主协议

      • 《汇聚支付-人民币支付服务协议》
      • 《汇聚支付-委托代付服务协议》

    • 补充协议

      • 《汇聚支付-人民币支付服务之补充协议》

  • 自助开通（自助开通产品的方式，需要在线生成电子版协议并支持在线下载或签署，使用的协议如下）

    • 主协议

      • 《汇聚支付-人民币支付服务协议》
      • 《汇聚支付-委托代付服务协议》

    • 补充协议

      • 《汇聚支付-人民币支付服务之补充协议》
      • 《汇聚支付-委托代付服务之补充协议》
2.3 关于协议签约

按照商户签约的产品类型以及签约顺序可以将签约性质进行以下分类

  • 首次签约：指商户进行某种主协议的第一次签署

    • eg. 商户第一次开通收单产品时，签署《汇聚支付-人民币支付服务协议》

  • 补充签约：指商户进行补充协议的签署

    • eg. 商户第二/三/…N次开通收单产品时，签署《汇聚支付-人民币支付服务之补充协议》

  • 自动续签：指汇聚根据主协议的自动续签条款，协议到期30天内如商户无异议则自动将协议续期的续签

  • 手动续签：指主协议中未签署自动续签条款时，商户与汇聚线下协商或商户主动在线续签等方式完成的续签

 

3、需求描述

3.1 优化商户入网运营流程
  • 商户通过产品码等自助开通方式的入网流程，优化销售人员填写和提交OA申请的环节，现只需销售或代理商将包含产品开通信息的产品码给到商户 ，商户在线填写产品码提交产品开通申请，后台审核后即可开通
  • 商户通过产品码等自助开通方式的入网流程，优化运营手工填写协议，手工开通产品的操作，现只需商户在线提交产品开通申请时，在线签署或下载打印并盖章寄回系统自动生成的服务协议，运营人员后台审核资质后，系统自动完成产品的开通
3.2 优化商户注册流程

  • 优化商户注册流程，划分为账户注册和实名认证两个独立流程

    • 缩短商户在线注册的体验环节，支持账户注册和实名认证分开完成
    • 解决商户资质到期的二次认证问题

  • 优化实名认证流程

    • 支持根据企业类型判断商户需要上传的证照资料
    • 支持商户修改和重新提交认证申请
    • 支持商户资质到期的二次认证申请

  • 优化用户体验界面

    • 汇聚官网提供注册入口
    • 商户后台提供邮件验证、账户创建和实名认证等服务
3.3 优化产品开通流程

  • 新增自助开通方式

    • 产品码开通：指定的产品和费率，可同时开通多个产品，通过代理商/销售获取产品开通码
    • 默认开通：默认的产品和费率
    • 普通开通：通用产品和费率，直接在线选择并提交开通申请

  • 新增产品开通申请表

    • 即一份详细的产品开通申请表，包含产品、费率、申请人、申请方式等申请信息
    • 商户无论通过哪种自助申请方式进行产品开通都需要提交一份申请单，作为汇聚审核和开通产品以及生成协议的凭证
    • 支持系统以申请单为凭证自动完成产品的开通

  • 优化用户体验界面

    • 产品中心新增“产品管理“>”我的产品”模块，商户可查看产品开通列表、协议有效期和费率等信息
    • 产品中心新增“产品管理”>“申请记录”模块，商户可查看自己的产品开通申请记录

    商户选择产品码开通的方式申请时，需要实现以下功能：

  • 新增产品模板

    • 即一张产品开通的列表，包含一个或多个产品及其手续费率等内容
    • 销售人员/代理商可以在各自的管理后台新建和维护产品模版

  • 新增产品码

    • 包含了产品开通列表、代理商/销售关系等信息
    • 商户可以在商户后台填写产品码来完成产品的自助开通
3.4 优化协议管理流程

  • 支持系统生成电子版协议

    • 商户通过自助开通方式的申请，系统可自动生成电子版协议
    • 如果商户是首次开通收款或付款产品，则自动生成《汇聚支付-人民币支付服务协议》或《汇聚支付-委托代付服务协议》，产品开通申请中包含跨协议的产品时，则生成2份服务协议
    • 如果商户非首次开通收款或付款产品，则自动生成《汇聚支付-人民币支付服务之补充协议》

  • 合同签约的线上化和标准化

    • 1.0版本：依然采用线下纸质协议签署和归档的流程

      • 将服务协议的手工填写环节系统化，提高运营效率
      • 产品开通遵循先启用收款产品，合同归档后再启用出款产品的规则

    • 2.0版本：接入第三方电子合同和电子签名服务认证平台

      • 解决电子合同的完整性、电子签名的有效性，以及电子合同的第三方存储等问题
      • 产品可在汇聚与商户完成在线签约协议的时候全部开通

2、业务逻辑

2.1 工作流原理

商户入网流程中，涉及到多步骤的审核流程，本次采用工作流引擎的原理来处理这块的业务逻辑，主要有以下两个流程：

  • 入网审核-自助开通
  • 入网审核-协助开通
2.1.1 业务逻辑

a.工作流程执行前需要完成初始化部署，涉及内容如下

  • 部署工作流程

    • 部署需要应用工作流的工作流程，例如“商户审核”
    • 详见 a.流程部署表 act_proc_deployment

  • 定义工作流程

    • 定义工作流程，1个流程部署可以定义多个工作流程，“商户审核”的流程定义如下

      • 入网审核-自助开通
      • 入网审核-协助开通
      • 过期认证
      • 新增产品

    • 详见 b.流程定义表 act_proc_definition

  • 设计工作流程

    • 设计每个工作流程的模型，记录工作流程中所有任务的先后顺序
    • 详见 c.流程设计模型表 act_proc_model

 

b.工作流程由发起人开始，执行过程以“入网审核-自助开通”为例

  • 开始工作流程

    发起人商户提交申请时，系统实例化工作流程，同时记录相关的流程执行信息

    • 创建实例化的工作流程，详见 d.流程执行实例表 act_proc_execution
    • 记录下一步执行的任务，详见 e.执行中流程任务记录表 act_ru_task
    • 记录已执行的历史任务，详见 f.执行流程历史任务记录表 act_hi_task
    • 记录任务相关的业务内容，详见 g.任务执行内容记录表 act_hi_task_record
    • 执行任务相关的业务操作，详见 2.2工作流应用：商户入网审核

  • 执行工作流程

    经办人执行工作流程中的任务，系统更新工作流程，同时记录相关的流程执行信息

    • 记录下一步执行的任务
    • 记录已执行的历史任务
    • 记录任务相关的业务内容
    • 执行任务相关的业务操作

  • 结束工作流程

    经办人执行结束任务，系统结束工作流程，同时记录相关的流程执行信息

    • 清空执行中流程的任务
    • 记录已执行的历史任务
    • 记录任务相关的业务内容
    • 执行任务相关的业务操作

2.2 工作流应用：商户入网审核
2.2.1 业务逻辑

1）工作流程定义

​ 创建账户时，记录商户选择的注册方式，商户进行实名认证时，根据注册方式执行相应的入网审核流程。现有商户入网审核的工作流程可定义为以下两种：

  • 入网审核-自助开通

  • 入网审核-协助开通

2）流程任务说明

a.商户提交申请

  • 工作流的起点，在商户成功提交实名认证时创建

  • 需要记录的内容如下：

    • 工作流相关：初始化工作流实例，记录执行任务记录

    • 工作流相关：记录商户提交的申请内容，包括产品开通和资质等信息，关联当前任务ID

      • 注：工作流执行中涉及到修改申请内容的任务，都需要单独记录该步骤的提交内容，关联当前任务ID
      • 注：涉及到修改申请内容的任务有：商户提交申请/商户修改申请/运营初审

    • 业务相关：创建业务申请单，并关联工作流执行ID，“业务申请单”详见 信息结构

      • 申请编号>>
      • 申请状态>> 审核中
      • 申请性质>> 首次认证
      • 流程执行ID>>

    • 账户相关：修改账户信息，详见 信息结构

      • 资质状态>> 认证中

b.商户修改申请

  • 商户修改资质信息，重新提交时创建

  • 需要记录的内容如下：

    • 工作流相关：记录执行任务记录

    • 工作流相关：记录修改后的申请内容，即资质信息，关联当前任务ID

    • 业务相关：修改业务申请单

      • 申请状态>> 审核中
      • 修改时间>> 当前时间

    • 账户相关：修改账户信息

      • 资质状态>> 认证中

c.运营初审

  • 运营审核资质，修改/补充信息，提交下一步时创建

  • 需要记录的内容如下：

    • 工作流相关：记录执行任务记录

    • 工作流相关：记录修改后的申请内容，即资质信息，关联当前任务ID

    • 工作流相关：记录运营驳回时填写的提示商户修改的信息，关联当前任务ID

    • 业务相关：修改业务申请单

      • 申请状态>> 已驳回（驳回时才修改）
      • 审核时间>> 当前时间

    • 账户相关：修改账户信息

      • 资质状态>> 待修改（驳回时修改）、认证失败（审核失败时修改）

d.运营复核

  • 运营复核，提交下一步时创建

  • 需要记录的内容如下：

    • 工作流相关：记录执行任务记录

    • 业务相关：修改业务申请单

      • 申请状态>> 审核通过（通过时才修改）
      • 审核时间>> 当前时间

    • 账户相关：修改账户信息

      • 资质信息>> 将上一个任务提交的申请内容从中间表更新到相应的账户信息表
      • 账户状态>> 已激活（审核通过时修改）
      • 资质状态>> 已认证（审核通过时修改）、认证失败（审核失败时修改）

e.销售设置等级（自助开通）

  • 销售后台进行的操作，提交下一步时创建

  • 需要记录的内容如下：

    • 工作流相关：记录执行任务记录

    • 账户相关：修改账户信息

      • 商户等级>> A/B/C

f.发送协议&开通产品（自助开通）

  • 自助开通时运营进行的操作，提交下一步时创建

  • 需要记录的内容如下：

    • 工作流相关：记录执行任务记录

    • 账户相关：修改账户信息，开通产品，生成合同

    • 当前时间大于等于有效期时，返回错误“产品码已过期”

  • 状态：未使用
  • 操作人：用户ID
  • 创建时间：当前时间

d.删除产品码

  • 删除方式：单个删除和批量删除
  • 选中一个或多个未使用的产品码，点击”删除“按钮，确认完成删除
  • 系统更新产品码记录和新增一条操作日志

4.4.3 运营开通产品

a.定义：商户提交产品开通申请后，运营在运营后台进行的产品开通审核

  • 申请进度

    • 当前进度：产品开通

  • 操作入口

    • 运营后台>>商户管理>>产品开通审核

  • 操作权限

    • 用户组：产品开通审核组

b.产品开通审核列表

  • 排序和筛选

    • 默认按申请时间倒序排列
    • 可按申请方式、申请状态、申请进度、申请人进行筛选

  • 操作

    • 产品开通：资料审核通过，根据产品开通申请进行产品开通
    • 审核不通过：资料审核不通过，结束申请流程

c.产品开通

  • 选择一条申请记录，并点击”审核“按钮，打开审核弹窗

  • 资料核实正确，点击“产品开通”按钮，填写审核意见，并确认提交，系统进行产品开通，同时更新申请记录和产品开通工作流的执行实例，并关闭当前弹窗

    • 产品开通

      • 收款产品状态：启用
      • 结算/付款产品状态：禁用
      • 商户编号：当前申请的商户编号

    • 更新产品开通申请记录：审核时间

    • 更新工作流程执行实例

      • 执行中任务记录表：修改当前任务记录
      • 历史执行任务记录表：增加一条任务名称为“产品开通”的记录
      • 历史变量记录表：增加1条记录，保存“产品开通”任务中提交的变量

d.审核不通过

  • 选择一条申请记录，并点击”审核“按钮，打开审核弹窗

  • 资料核实有误，点击“审核不通过”按钮，填写审核意见，并确认提交，系统更新申请记录，同时更新产品开通工作流的执行实例，并关闭当前弹窗

    • 更新商户产品开通的申请记录：

      • 申请状态：审核中>>审核不通过
      • 审核时间：当前时间

    • 更新工作流程执行实例

      • 执行中任务记录表：删除当前任务记录
      • 历史执行任务记录表：增加一条任务名称为“产品开通”的记录
      • 历史变量记录表：增加1条记录，保存“产品开通”任务中提交的变量

e.如果运营选择了产品开通，工作流的当前进度转到“协议归档”任务，详见4.5.1运营管理合同
