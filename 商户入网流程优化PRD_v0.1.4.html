一、需求概述

1、需求背景

 
xxxxxxxxxx
   当前商户入网开通流程中，销售人员需要先收集商户的证照等资料和产品开通需求，然后提OA入网流程（填写商户资料和产品开通费率等信息），运营人员根据OA申请进行资质审核并手动填写纸质版协议后，再为商户开通产品和激活账户，期间纸质协议会在运营人员、销售人员和商户之间往来邮寄多次，商户的产品开通顺序也受到协议签署结果的影响，一般需等纸质协议归档后才能使用结算功能。
   针对以上问题，本次商户入网流程的优化主要集中在资料收集、产品开通、协议签署以及OA审核等环节，达到简化运营流程、平衡和减轻运营和销售人员工作量、实现商户入网签约流程标准化和线上化、提高入网签约效率的目的。

 

2、需求描述

2.1 优化商户入网运营流程

  • 针对通过产品开通码等自助申请方式的商户入网签约，优化销售人员收集商户产品开通要求，再填写和提交OA申请的环节，现只需商户在线提交产品开通申请，销售部在销售后台审核申请（产品、费率等），审核后提交给运营开通产品
  • 通过自助申请方式的产品开通，优化运营人员手动填写协议，再邮寄和归档协议的环节，现只需商户提交产品开通申请，销售部审核申请，运营人员通过系统进行产品开通和协议生成，商户只需登录商户后台进行协议的在线确认签署即可完成产品的开通。

2.2 优化商户在线注册流程

  • 优化商户注册流程，划分为账户注册和实名认证两个独立流程

    • 缩短商户在线注册的体验环节，支持账户注册和实名认证分开完成
    • 解决商户资质到期的二次认证问题

  • 优化实名认证流程

    • 支持根据企业类型判断商户需要上传的证照资料
    • 支持商户修改和重新提交认证申请
    • 支持商户资质到期的二次认证申请

  • 优化用户体验界面

    • 汇聚官网作为注册入口，提供账户名申请和邮件验证服务
    • 商户后台提供账户创建和实名认证服务

2.3 优化产品开通流程

  • 新增自助开通方式

    • 产品码开通：指定的产品和费率，可同时开通多个产品，通过代理商/销售获取产品开通码
    • 普通开通（暂无）：通用产品和费率，直接在线选择并提交开通申请

  • 新增产品开通申请单

    • 即一份详细的产品开通申请表格，包含费率、申请人、申请方式和申请时间等信息
    • 商户无论通过哪种自助申请方式进行产品开通都需要提交一份申请单，作为汇聚审核和开通以及生成服务协议的凭证

  • 优化用户体验界面

    • 在产品中心新增“产品管理>我的产品”模块，商户可按照产品类型查看产品开通情况、协议有效期和费率等信息

    商户选择产品码开通的方式申请时，需要实现以下功能：

  • 新增产品开通模板

    • 即一张产品开通的列表，包含一个或多个产品及其手续费率等内容
    • 销售人员/代理商可以在各自的管理后台新建和维护产品开通模版
    • 一个产品开通模板可以生成多个产品开通码，提供给商户用于自助开通产品

  • 新增产品开通码

    • 关联了产品开通模板、代理商/销售人员等信息
    • 商户可以在商户后台填写产品开通码来提交产品开通申请

2.4 优化协议管理流程

  • 支持产品开通申请单自动生成电子协议

    • 如果商户是首次开通收单或代付产品，则自动生成《汇聚支付-人民币支付服务协议》或《汇聚支付-委托代付服务协议》
    • 如果商户已经开通过产品，则自动生成《汇聚支付-人民币支付服务之补充协议》或《汇聚支付-委托代付服务之补充协议》
    • 产品开通申请单中包含跨协议的产品时，则生成2份服务协议

  • 支持商户在商户后台进行协议的查看、下载、在线签约和续签等操作

  • 接入第三方电子合同服务平台

    • 满足合规性要求，保证电子合同的完整性、电子签名的有效性，解决合同的存放等问题

 

3、需求定义

3.1 关于产品开通

  • 商户入网签约到开通产品的整个体验过程可以分为

    • 账户注册
    • 实名认证
    • 产品开通

  • 产品开通也是一个独立的流程，产品的开通方式可以划分为以下几种

    • 协助申请：销售发起申请，提OA附带产品申请信息，运营审核开通，签署纸质版/电子版协议

    • 自助申请：商户发起申请，提交产品开通申请单，销售审核费率，运营审核开通，签署电子版协议

      • 产品码开通：指定的产品和费率，可同时开通多个产品，通过代理商/销售获取产品开通码
      • 普通开通：通用产品和费率，直接在线选择并提交开通申请

3.2 关于协议分类

  • 协助开通（目前公司的产品开通方式，使用的协议如下）

    • 主协议

      • 《汇聚支付-人民币支付服务协议》
      • 《汇聚支付-委托代付服务协议》

    • 补充协议

      • 《汇聚支付-人民币支付服务之补充协议》

  • 自助开通（自助开通产品的方式，需要在线生成电子版协议并支持在线签署，建议使用的协议如下）

    • 主协议

      • 《汇聚支付-人民币支付服务协议》
      • 《汇聚支付-委托代付服务协议》

    • 补充协议

      • 《汇聚支付-人民币支付服务之补充协议》
      • 《汇聚支付-委托代付服务之补充协议》

3.3 关于协议签约

按照商户签约的产品类型以及签约顺序可以将签约性质进行以下分类

  • 首次签约：指商户进行某种主协议的第一次签署

    • eg. 商户第一次开通收单产品时，签署《汇聚支付-人民币支付服务协议》

  • 补充签约：指商户进行补充协议的签署

    • eg. 商户第二/三/…N次开通收单产品时，签署《汇聚支付-人民币支付服务之补充协议》

  • 自动续签：指汇聚根据主协议的自动续签条款，协议到期30天内如商户无异议则自动将协议续期的续签

  • 手动续签：指主协议中未签署自动续签条款时，商户与汇聚线下协商或商户主动在线续签等方式完成的续签

3、功能详述

3.1 账户注册

a.注册入口在汇聚官网，用户点击后可选择注册账户类型，有“企业账户”和“个人账户”两种选择；

b.用户点击“创建企业账户”时，需在弹窗中点击同意服务协议才能进入注册页面，否则返回当前页面；

c.页面填写内容要求：

d.用户点击“发送验证邮件”按钮，检验输入内容，系统发送验证邮件到该用户填写的邮箱

  • 记录保存到tbl_boss_email_verify
  • 同个邮箱地址请求发送验证邮件的时间间隔为60秒
  • 注册链接有效期为30分钟
  • 邮件内容同原模板
  • 按钮点击请求成功后，页面进入下一步，“重新发送按钮”刷新倒计时60秒

e.用户点击“重新发送按钮”返回上一步，邮箱地址输入框自动填充

 

3.2 账户创建

a.用户点击注册链接进入商户后台的“账户注册”页面，页面上显示用户注册的账户名

b.页面填写内容要求：

c.用户点击“获取验证码”按钮，检验手机号码是否唯一且有效后，下发短信验证码到该手机号

  • 验证码有效期为10分钟
  • 同一手机请求发送验证码的时间间隔为60秒
  • 按钮每次点击请求成功后，显示倒计时为60秒，状态变为禁用，倒计时结束恢复启用状态

d.用户点击“下一步”按钮，检验输入内容 ，完成账户和管理员的创建

  • 账户信息记录到tbl_user_info，重要的字段如下：

    • 账户名：商户注册时使用的邮箱，不支持修改
    • 账户编号（商户号）：888+企业类型+行业类型+序号+代理商编号+随机码
    • 安全绑定手机：商户注册时使用的手机号码，支持修改
    • 安全绑定邮箱：商户注册时使用的邮箱，支持修改
    • 支付密码：用户交易时使用，支持修改

  • 管理员信息记录到tbl_portal_user_operator，重要的字段如下：

    • 登录账号：商户注册时使用的邮箱，不支持修改
    • 登录密码：用于登录商户后台，支持修改
    • 联系手机：商户注册时使用的手机，支持修改

e.关于账户状态的定义

  • 主状态：指账户的使用状态，限制账户功能和交易功能的使用
  • 资质状态：指账户的资质认证状态，与主状态一起限制交易功能的使用
  • 与资金账户状态的区别：资金账户状态只限制资金的出入账

f.账户创建成功后，实现管理员自动登录到商户后台

  • 账户完成创建后，主状态为已创建，资质状态为未认证

  • 菜单权限

    • 首页：异常页面
    • 交易中心：全部二三级菜单
    • 产品中心：全部二三级菜单
    • 商户中心：“用户信息”-全部 + “商户信息”-安全设置/签约管理

  • 操作权限

    • 管理员

      • 实名认证申请
      • 产品开通申请

    • 操作员

      • 无

 

3.3 实名认证

3.3.1 商户首次实名认证

初次提交申请

a.定义：商户完成账户创建后，第一次提交实名认证申请

  • 账户状态

    • 主状态：已创建
    • 资质状态：未认证

  • 操作入口

    • 账户创建成功页面
    • 商户后台>>首页
    • 商户后台>>商户中心>>安全设置>>实名认证

  • 操作权限

    • 管理员

b.用户进入实名认证页面，选择企业类型，点击下一步进入提交证件信息页面

  • ”下一步“按钮默认为禁用状态，当选择了企业类型时，变为启用状态

c.提交证件信息页，页面填写内容要求如下

  • 企业类型：企业
  • 企业类型：事业单位/社会团体
  • 企业类型：个体工商户
  • 企业类型：自然人

d.用户点击“提交”按钮，先校验输入内容，通过校验后才弹出确认框

  • 点击“返回修改”按钮时，关闭当前弹窗

e.点击“继续提交”按钮时，系统创建一条实名认证的申请记录，同时初始化一条实名认证工作流的执行实例，并转到提交成功页面

  • 更新账户的资质状态

    • 未认证>>认证中

  • 创建商户实名认证的申请记录，详细字段见本节4.2实名认证

    • 申请状态：审核中
    • 申请性质：首次认证

  • 初始化一条实名认证工作流的执行实例，将用户提交的内容保存到历史变量记录表

    • 流程执行表：创建一条新的记录
    • 执行中任务记录表：创建一条任务名称为“提交申请”的记录
    • 历史执行任务记录表：创建一条任务名称为“提交申请”的记录
    • 历史变量记录表：创建多条记录，保存“提交申请”任务中用户填写的所有变量

  • 关于工作流的定义，详见本节4.1工作流程

f.提交成功页面，用户可以在页面点击“开通产品”或“查看进度”

  • 首次实名认证申请提交后，资质状态为认证中

  • 菜单权限

    • 首页：异常页面
    • 交易中心：全部二三级菜单
    • 产品中心：全部二三级菜单
    • 商户中心：“用户信息”-全部 + “商户信息”-安全设置/签约管理

  • 操作权限

    • 管理员

      • 实名认证申请
      • 产品开通申请

    • 操作员

      • 无

 

修改后提交申请

a.定义：商户完成账户创建后，提交首次实名认证申请且申请被驳回，商户需修改申请后重新提交

  • 账户状态

    • 主状态：已创建
    • 资质状态：认证中

  • 申请进度

    • 申请状态：已驳回
    • 工作流进度：修改申请

  • 操作入口

    • “申请进度”页面点击“查看原因”按钮，在弹出框中点击“前往修改”按钮

  • 操作权限

    • 管理员

b.用户进入修改企业类型的页面，加载用户上一次提交的历史变量内容

c.用户点击“提交”按钮，先校验输入内容，通过校验后才弹出确认框

  • 点击“返回修改”按钮时，关闭当前弹窗

d.点击“继续提交”按钮时，系统更新申请记录，同时更新实名认证工作流的执行实例，并转到提交成功页面

  • 更新商户实名认证的申请记录 APPLY_CERTIFICATION_MERCHANT

    • 更新状态：已驳回>>审核中
    • 更新修改时间：当前时间

  • 更新工作流程执行实例，将用户本次提交的内容保存到历史变量记录表

    • 执行中任务记录表：修改当前任务记录
    • 历史执行任务记录表：增加一条任务名称为“修改申请”的记录
    • 历史变量记录表：增加多条记录，保存“修改申请”任务中用户填写的所有变量值

f.提交成功页面，用户可以在页面上点击“查看进度”

 

3.3.2 商户资质过期重新实名认证

初级提交申请

a.定义：商户资质即将到期的1个月内或资质已过期后，商户第一次提交实名认证申请

  • 账户状态

    • 主状态：已激活（已被冻结的账户无法登录商户后台，因此无法申请实名认证）
    • 资质状态：已认证或已过期

  • 操作入口

    • 资质即将过期时：管理员登录商户后台，点击首页顶部的提示按钮
    • 资质已过期时：管理员登录商户后台，点击首页的“重新认证”按钮

  • 操作权限

    • 管理员

b.用户进入实名认证页面，重新上传营业执照，修改营业期限

c.用户点击“提交”按钮，先校验输入内容，通过校验后才弹出确认框

  • 点击“返回修改”按钮时，关闭当前弹窗

d.点击“继续提交”按钮时，系统创建一条实名认证的申请记录，同时初始化一条实名认证工作流的执行实例，并转到提交成功页面

  • 更新账户的资质状态

    • 不需要更新

  • 创建商户实名认证的申请记录，详细字段见本节4.2

    • 申请状态：审核中
    • 申请性质：过期认证

  • 初始化一条实名认证工作流的执行实例，将用户提交的内容保存到历史变量记录表

    • 流程执行表：创建一条新的记录
    • 执行中任务记录表：创建一条任务名称为“提交申请”的记录
    • 历史执行任务记录表：创建一条任务名称为“提交申请”的记录
    • 历史变量记录表：创建多条记录，保存“提交申请”任务中用户填写的所有变量值

f.提交成功页面，用户可以在页面上点击“查看进度”

  • 商户的资质过期后，资质状态为已过期

  • 菜单权限

    • 首页：异常页面
    • 交易中心：全部二三级菜单
    • 产品中心：全部二三级菜单
    • 商户中心：全部二三级菜单

  • 操作权限

    • 管理员

      • 账户功能：可用
      • 交易功能：禁用

    • 操作员

      • 无

 

修改后提交申请

a.定义：商户资质即将到期或已过期后，商户重新提交的实名认证申请被驳回，商户需修改申请后重新提交

  • 账户状态

    • 主状态：已激活
    • 资质状态：已认证或已过期

    • 更新工作流程执行实例

      • 执行中任务记录表：修改当前任务记录
      • 历史执行任务记录表：增加一条任务名称为“产品开通”的记录
      • 历史变量记录表：增加1条记录，保存“产品开通”任务中提交的变量

d.审核不通过

  • 选择一条申请记录，并点击”审核“按钮，打开审核弹窗

  • 资料核实有误，点击“审核不通过”按钮，填写审核意见，并确认提交，系统更新申请记录，同时更新产品开通工作流的执行实例，并关闭当前弹窗

    • 更新商户产品开通的申请记录：

      • 申请状态：审核中>>审核不通过
      • 审核时间：当前时间

    • 更新工作流程执行实例

      • 执行中任务记录表：删除当前任务记录
      • 历史执行任务记录表：增加一条任务名称为“产品开通”的记录
      • 历史变量记录表：增加1条记录，保存“产品开通”任务中提交的变量

e.如果运营选择了产品开通，工作流的当前进度转到“生成协议”任务，详见3.5.1运营管理合同
